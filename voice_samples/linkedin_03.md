# Type: Project showcase + learnings post

Over the past few weeks, I have been using different tools to test and build early versions of my ideas. Here's one I had a lot of fun creating:

Word Chaos - it is an interactive AI storytelling tool where users can evolve a story by adding new words as prompts. I built this using Replit, and ChatGPT, experimenting with how AI can generate immersive stories in real-time.

Here's what I've learned from all the experimenting.

What I loved:
- Super beginner-friendly: you just describe what you want to build in english and within a few minutes you can see a working prototype.
- No coding background needed: You don't have to "learn to code" before starting, you can just start right away.
- Rapid prototyping: The speed at which ideas come to life is incredible. You just need to explain what you want clearly.

What I learned (the hard way):
- Clarity matters: If you are not concise, the tools can go in circles or generate confusing results.
- Debugging takes time: Making the first iteration is easy but debugging is what takes most of the effort. Keep things simple and go iteratively similar to how you would go about launching a product in phases.
- Collaborate with different tools: if you are stuck, try asking different LLMs or pair tools to help debug and refine. My goto models are ChatGPT and Claude.

But the biggest learning? You don't need a perfect team, perfect stack, or perfect timing to test an idea. You just need curiosity and a place to start.
